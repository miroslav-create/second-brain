## Decisions

### Notion private Kanban — schema (minimal)
- DB in user's private space inside `pojistovnadirect` workspace.
- Properties:
  - **Status** (select, the Kanban column): `Inbox | Today | Doing | Waiting | Done | Archive`
  - **Type** (select): `delivery | code | learn | admin | people | decision`
  - **Due** (date)
- Bridge to Obsidian: free-form link/URL in task description body when a breakdown note exists. No formal relation property.
- Views: `Board by Status` (default), `Today`, `Inbox`, `By Type`.
- **NOT on board**: pure reference notes (→ Obsidian), recurring routines (→ calendar/habit), team-tracked Fondee tickets in public DEV/Product boards (link only, never duplicate).

### Obsidian — role & structure
Ranked roles: **Daily note (capture)** > **Evergreen Zettel** > **Graph view = mind map** > **Canvas (rare)**.

Folder layout inside `vault\`:
```
vault\
├── daily\              ← YYYY-MM-DD.md, has ## Inbox section
├── people\             ← one note per colleague/contact
├── projects\           ← project notes (own + observed)
├── evergreen\          ← atomic concept notes, freely linked
├── meetings\           ← long-form reasoning (bullet outcomes go to Notion meetings DB)
├── reference\          ← how-to / personal docs
├── canvas\             ← .canvas files for spatial diagrams
├── _archive\           ← old vault, read-only, excluded from graph
└── _templates\         ← Templater templates
```

Plugins day-1: **Obsidian Git** (auto-commit hourly + push), **Templater** (daily / evergreen / meeting / project templates), core **Daily Notes** + **Templates**. Defer Dataview until felt missing.

Overlap rule with Notion (sharp): **state machine → Notion. Prose + links → Obsidian.** Meetings = bullets to Notion Meetings DB, long reasoning to Obsidian, link via `[[meeting-YYYY-MM-DD-topic]]`.

### Capture inbox — mixed by context, on-demand triage
| Context | Capture target |
|--|--|
| Desktop work hours | Notion private Kanban Inbox (one-line row) |
| Sit-down thinking / journaling | Obsidian daily note `## Inbox` section |
| Phone / walk / shower | Typed into Notion mobile (work) or Obsidian mobile (thinking). No voice. |

Triage = on-demand only (`/triage`). No fixed time, no cron, no nag. User runs when he chooses.

### Claude Code — scope
- **Working dir**: `Projects\second-brain\` (sibling to other coding projects).
- **Owns**: triage, sparring (`/grill-with-docs`), drafting (specs/emails/Notion pages from bullets), cross-tool search, pushing Obsidian breakdowns → Notion tasks.
- **You own**: final wording, decisions, code commits, anything sent externally.
- **No fixed daily/weekly rituals.** Invoked at user's rhythm.

### Vault sync / backup
- **OneDrive** auto-sync (inherited from `OneDrive - Direct\Projects\` path) — device sync layer.
- **Private GitHub repo** `second-brain` — version history, diff, rollback. Obsidian Git plugin auto-commits hourly + pushes.
- `.gitignore`: `.obsidian/workspace*`, `.obsidian/cache`, large binaries, `inbox-cache\`.

### MCPs / integrations
- **Notion MCP**: already in deferred-tools list (`mcp__claude_ai_Notion__*`). Authenticate on first use (`notion-search`, `notion-fetch`, `notion-create-pages`, `notion-update-page`, etc.).
- **Obsidian**: filesystem-only — Read/Edit/Glob/Grep over `vault\`. No special MCP needed.
- **No voice / transcription** pipeline.

### MVP skills (build in `.claude/skills/`)
1. **`/triage`** — read Obsidian daily inbox (today's daily note `## Inbox`) + Notion private Kanban `Inbox` rows → propose per item: Notion task (which Type, Due?), evergreen note, discard. User confirms batch.
2. **`/spar`** — wrap existing `/grill-with-docs` for personal decisions/designs. Updates `CONTEXT.md` as terms resolve.
3. **`/draft`** — bullets/notes → Notion page, Slack/email message, or spec section. User edits final.
